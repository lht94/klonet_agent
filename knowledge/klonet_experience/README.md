# Klonet 经验知识层

本层把原始操作记录和规范知识转换为可复用的诊断案例。案例不是历史事故结论；除非有日志或代码证据，否则根因均标记为“候选”，并给出确认标准。

## 使用方式

1. 按现象选择案例。
2. 从调用链上游向下游收集证据。
3. 满足确认标准后再执行修复。
4. 修复后完成最小业务闭环，并补充实际根因、环境和日期。
5. 地址、账号、口令、令牌和内部域名必须脱敏。

## 目录

- `common_errors.md`：现象到案例的入口。
- `ops_cases.md`：运行、部署与基础设施案例索引。
- `dev_cases.md`：前后端开发定位案例索引。
- `cases/`：标准化诊断手册。
