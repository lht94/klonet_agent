# Klonet Agent 常见任务模板

## Mentor 问答

适用场景：解释 Klonet 架构、源码、报错、开发流程。

推荐流程：

1. 调用 `search_knowledge` 检索证据。
2. 必要时调用 `read_project_journal` 查看当前项目进度。
3. 说明依据、结论和不确定点。
4. 给出下一步学习或排查建议。

## Coding 开发

适用场景：修改 workspace 内代码、补测试、生成 diff、记录日志。

推荐流程：

1. 调用 `update_todos` 拆解任务。
2. 调用 `search_knowledge` 查规范和相似实现。
3. 使用 `list_files`、`read_file` 阅读相关文件。
4. 使用 `write_file` 做最小必要修改。
5. 使用 `run_tests` 验证。
6. 使用 `show_diff` 检查改动。
7. 使用项目日志工具记录执行过程、测试结果和验收差异。

## 报错排查

适用场景：用户给出异常、测试失败或运行失败。

推荐流程：

1. 先复述错误现象和运行环境。
2. 检索知识库中的相似问题。
3. 定位报错属于导入、路径、依赖、测试、权限还是业务逻辑。
4. 给出最小验证命令。
5. 如果证据不足，明确说明还需要读取哪些文件或日志。

## 修复测试失败

适用场景：已有测试失败，需要 Coding Agent 做最小修复。

推荐流程：

1. 读取失败信息，确认失败测试名称、报错文件和错误行。
2. 检索知识库，查找相似实现和项目规范。
3. 读取相关源码和测试文件。
4. 只做解决当前失败所需的最小修改。
5. 重新运行失败测试；通过后再运行相关测试集。
6. 查看 diff，确认没有无关改动。
7. 将失败原因、修复方式和验证结果写入项目日志。
