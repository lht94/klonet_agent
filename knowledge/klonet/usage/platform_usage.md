# Klonet 平台普通用户使用指南

## 适用场景

用户已经有可访问的 Klonet Web 平台地址，想通过浏览器使用 Klonet 做网络仿真、教学实验或拓扑实验。

这种问题不是首次安装环境，也不是启动服务器上的 Master、Worker、Redis、Celery 或 Nginx。普通用户通常只需要浏览器；如果要登录实验节点，可能还需要平台提供的 Web Terminal、SSH 信息或文件上传下载入口。

## 使用前准备

1. 确认管理员已经给出平台访问地址、账号和密码。
2. 使用浏览器打开平台地址。
3. 如果页面打不开，先记录浏览器地址、报错和时间，再让管理员检查平台服务和 Nginx。
4. 如果能打开页面但无法登录，先记录用户名、请求时间和浏览器 Network 中的登录请求结果。

## 基本实验流程

1. 登录 Klonet Web 平台。
2. 进入拓扑或实验相关页面。
3. 新建或导入实验拓扑。
4. 按实验需求添加主机、交换机、路由器、控制器、KVM 节点或卫星相关节点。
5. 配置节点参数、链路参数和实验镜像。
6. 提交拓扑创建或部署请求。
7. 等待进度完成，观察页面状态、拓扑状态和实验数据。
8. 需要进入节点时，优先使用平台提供的 Web Terminal 或 SSH 入口。
9. 实验结束后删除拓扑，避免资源残留。

## 做实验时要区分的角色

- 操作者电脑：用户手里的笔记本或台式机，一般只负责打开浏览器、上传文件、下载结果。
- Klonet 平台服务器：运行 Master、Worker、Redis、MySQL、Celery 和 Nginx 的机器。
- 实验节点：平台创建出来的容器、KVM 虚机、交换机、路由器或控制器。

用户问“我电脑要下载什么”“怎么用 Klonet 做实验”时，默认站在操作者电脑和浏览器使用视角，不要回答服务器依赖安装流程。

## 常见分支

### 只是想开始使用平台

回答应先给浏览器使用路径：打开平台地址、登录、进入实验或拓扑页面、新建或导入拓扑、部署、观察进度、进入节点、删除拓扑。

### 想知道个人电脑要装什么

默认只需要浏览器。可选工具包括 SSH 客户端、SCP/SFTP 工具、VS Code、数据库客户端或抓包工具，但这些不是普通浏览器使用平台的前置条件。

### 平台还没有访问地址

这通常不是普通用户使用问题，而是平台部署或平台启动问题。需要先确认是首次环境安装，还是已有环境的服务启动。

### 拓扑部署卡住

这属于排障问题，应转向拓扑进度、Celery、Redis、Worker 和资源残留检查，而不是重新讲普通用户入门流程。
