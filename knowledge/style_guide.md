# Klonet Agent 代码与注释风格指南

## 基本原则

- 优先沿用现有模块边界，不为了抽象而抽象。
- 代码要适合同学学习，命名直观，流程清楚。
- 中文注释用于解释模块职责、设计原因和容易踩坑的位置。
- 避免机械注释，例如“给变量赋值”这类不提供新信息的注释。

## 模块边界

- `llm/` 只负责模型调用。
- `tools/` 负责模型可调用工具和工具执行分发。
- `memory/` 负责长期记忆、用户画像和对话历史。
- `journal/` 负责项目 Markdown 状态机。
- `knowledge/` 负责 Klonet 文档、规范、源码和项目经验检索。
- `workspace/` 负责用户项目副本、路径限制和命令安全。

## 测试与验证

- 修改公共逻辑后至少运行导入测试和相关单元测试。
- Coding Agent 完成开发任务后要记录测试命令、测试结果和仍未覆盖的风险。
