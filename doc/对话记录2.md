### Klonet 源码架构（确认版）

#### 1. 入口层 — `mains/`
| 文件 | 角色与职责 |
| :--- | :--- |
| `master_main.py` | Master 主进程入口，初始化并启动管理端 Web 服务 |
| `worker_main.py` | Worker 主进程入口，承载各宿主机节点虚拟化控制服务 |
| `celery_worker.py` | Celery 异步任务 Worker 节点进程 |
| `web_terminal_main.py` | Web 终端服务，提供浏览器端的底层容器 SSH 访问 |
| `gun.py` / `worker_gun.py` | Gunicorn 生产级 WSGI 启动配置文件 |
| `register_admin.py` / `register_worker.py` | 管理员/Worker 服务节点动态注册脚本 |

#### 2. API 与 Web 层 — `webserver/`
| 路径 | 职责说明 |
| :--- | :--- |
| `app_factory.py` | Flask 应用工厂，负责全局蓝图（Blueprint）和扩展组件注册 |
| `api/` | 所有标准 REST 路由（包含拓扑动态修改、用户认证、镜像管理等） |
| `tasks/` | Celery 异步长任务实现（其中拓扑部署 `tasks/topo/tasks.py` 为核心） |
| `socketio_handlers.py` | WebSocket 实时推送处理器，向前端同步部署进度与系统日志 |
| `models/` / `schema/` | 数据 ORM 模型映射 / 传入请求体严苛字段校验 |

#### 3. 服务层 — `Service_layer/`
| 文件 | 职责说明 |
| :--- | :--- |
| `redisAPI.py` | Redis 缓存状态、动态拓扑元数据与异步进度表读写封装 |
| `mysql_manager.py` + `mysql_api/` | MySQL 关系型数据库连接池与基础 CRUD 访问封装 |
| `NEManager.py` | 网元（虚拟主机节点、网络链路）全生命周期管理业务抽象 |
| `AsyncTopoManager.py` | 异步拓扑部署流水线编排与控制 |
| `TopoManager.py` | 静态拓扑结构的语义 CRUD |
| `LinkManager.py` | 虚拟网络链路复杂的业务逻辑控制 |

#### 4. 编排层 — `Function_layer/`
| 文件 | 职责说明 |
| :--- | :--- |
| `topo_preprocess.py` | 前端拓扑描述 JSON 校验、版本兼容性转换与子网切分 |
| `resource_manager.py` | 集群资源开销评估与多 Worker 节点最优化选择调度算法 |
| `deploy_process_bar.py` | 高频部署过程中的 Redis 状态机进度条细粒度写入 |

#### 5. 实现层 — `Implement_layer/`
| 目录 | 底层具体技术实现 |
| :--- | :--- |
| `ContainerManager/` | 调用 Docker SDK/CLI 执行容器节点创建、生命周期与状态控制 |
| `LinkManager/` | 基于 Open vSwitch (OVS) 网桥、veth pair 及 Linux tc 工具限速 |
| `NE_Management/` | 虚机与高精度网元底层操作（结合 virsh、Linux bridge 等内核控制） |
| `ImageRegistryManager/` | 私有 Registry 镜像安全拉取与版本注册控制 |
| `DockerSwarmManager/` | 跨主机容器编排底座 Docker Swarm 集群管理 |

#### 核心调用链（以拓扑部署为例）
```text
前端请求 → api/topo/worker_topo.py
         → Celery: tasks/topo/tasks.py (master_deploy_topo)
              → Function_layer/topo_preprocess.py   （校验 + 切分）
              → Function_layer/resource_manager.py  （选 Worker）
              → Service_layer/redisAPI.py           （写进度表）
              → 分发 HTTP 到各 Worker
                   → Service_layer/NEManager.py     （业务编排）
                        → Implement_layer/ContainerManager/  （建容器）
                        → Implement_layer/LinkManager/       （建链路）
              → Function_layer/deploy_process_bar.py（更新进度）
